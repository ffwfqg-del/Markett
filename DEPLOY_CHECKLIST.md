# ✅ Чеклист для деплоя на Vercel

## Перед деплоем

- [ ] Все секреты удалены из кода (токены, API ключи)
- [ ] `.env` файлы добавлены в `.gitignore`
- [ ] `scripts/settings.json` не содержит реальных токенов
- [ ] Все зависимости указаны в `package.json`
- [ ] Код закоммичен и запушен в GitHub

## Настройка Vercel

- [ ] Создан аккаунт на Vercel
- [ ] Подключен GitHub репозиторий
- [ ] Проект создан в Vercel
- [ ] Framework определился как Next.js

## Переменные окружения

- [ ] `NEXT_PUBLIC_BOT_TOKEN` - токен бота
- [ ] `NEXT_PUBLIC_BOT_USERNAME` - username бота
- [ ] `NEXT_PUBLIC_API_URL` - URL приложения (обновить после деплоя!)
- [ ] `API_ID` - Telegram API ID
- [ ] `API_HASH` - Telegram API Hash
- [ ] `ADMIN_IDS` - ID админов
- [ ] `LOG_BOT_TOKEN` - (опционально) токен бота для логов

## После деплоя

- [ ] Скопирован URL приложения
- [ ] Обновлен `NEXT_PUBLIC_API_URL` на Vercel URL
- [ ] Выполнен Redeploy
- [ ] Приложение открывается без ошибок
- [ ] API endpoints работают (`/api/telegram/...`)
- [ ] Telegram Web App открывается
- [ ] Нет ошибок в логах Vercel

## Проверка работы

- [ ] Главная страница загружается
- [ ] Авторизация работает
- [ ] API запросы проходят успешно
- [ ] Нет ошибок в консоли браузера
- [ ] Нет ошибок в логах Vercel

## Важно помнить

- ⚠️ Python бот (`scripts/`) НЕ работает на Vercel - нужен отдельный сервер
- ⚠️ SQLite база данных НЕ работает на Vercel - нужна внешняя БД
- ⚠️ Сессии бота должны храниться на сервере, где работает бот
- ⚠️ Все секреты только в переменных окружения Vercel
